骚年-Markdown/Oracle/组件安装/11G-XMLDB组件安装-2020年4月7日

==================================================================

11G-XMLDB组件安装

==================================================================

0.查看数据文件情况

set pagasize 50000

set line 500

col file_name for a50

select tablespace_name,file_name from dba_data_files;

Oracle查询temp表空间的名字和位置

select tablespace_name,file_name from dba_temp_files;

以下操作一定要在 dba权限进行操作。

1.首先一定要确定有表空间“XMLDB”，若没有新建表空间

create tablespace XMLDB datafile ‘/oradata/sadb/XMLDB.dbf’ size 200M autoextend on;

2.接着 要新建XDB的user，并授予权限。

create user XDB

identified by XDB;

GRANT SELECT ANY TABLE TO XDB;

3.执行安装脚本安装

select tablespace_name,file_name from dba_temp_files;

set echo on

spool xdb_install.log

@?/rdbms/admin/catqm.sql xdb_user_pass xdb_tbs temp_tbs

编译失效对

@?/rdbms/admin/utlrp.sql

spool off;

show parameter name

show parameter dispatchers

alter system set dispatchers=”(PROTOCOL=TCP)(SERVICE=SADB)”;

4.重启数据库

shutdown immediate；

startup；

5.测试是否安装成功

select comp_name, status, version from DBA_REGISTRY where comp_name= ‘Oracle XML Database’;

验证是否安装成功：

set pagesize 1000

col comp_name format a36

col version format a12

col status format a8

col owner format a12

col object_name format a35

col name format a25

select comp_name, version, status from dba_registry where comp_id = ‘XDB’;

查询失效对象

select owner, object_name, object_type, status

from dba_objects

where status = ‘INVALID’

and owner in (‘SYS’, ‘XDB’);

查询DG状态

set pagesize 20

select process,status from v$managed_standby;

ALTER DATABASE RECOVER MANAGED STANDBY DATABASE NODELAY PARALLEL 2 USING CURRENT LOGFILE DISCONNECT FROM SESSION;

alter system switch logfile;

select thread#,max(sequence#) from v$log_history group by thread#;

archive log list
